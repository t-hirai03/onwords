# 導入事例ページ ドキュメント

本番サイト（https://www.onwords.co.jp/case/）から抽出したHTML構造とCSSスタイルを記録したドキュメント群です。

## 目的

WordPressテーマで導入事例一覧ページを実装する際に、本番サイト（STUDIO）と完全に同じ見た目を再現するための参考資料として使用します。

## ディレクトリ構造

```
.docs/case/
├── README.md          # このファイル
├── mv/                # MVセクション
│   ├── html.md        # HTML構造
│   └── css.md         # CSSスタイル（全ブレークポイント対応）
├── navigation/        # ナビゲーションセクション
│   ├── html.md        # HTML構造
│   └── css.md         # CSSスタイル（全ブレークポイント対応）
└── cards/             # カードセクション
    ├── html.md        # HTML構造
    └── css.md         # CSSスタイル（全ブレークポイント対応）
```

## 各セクションの説明

### 1. MVセクション（mv/）

ページ上部のヒーローセクション

- **html.md**: 背景画像・ラベル・タイトルを含むHTML構造
- **css.md**: PC・タブレット・スマホ全てのブレークポイントに対応したスタイル

**主な要素:**
- 背景画像（::before疑似要素）
- "Case" ラベル
- "導入事例" タイトル
- オーバーレイ

### 2. ナビゲーションセクション（navigation/）

カテゴリーフィルターナビゲーション

- **html.md**: カテゴリーリンクを含むHTML構造
- **css.md**: PC・タブレット・スマホ全てのブレークポイントに対応したスタイル

**主な要素:**
- "すべて" ボタン（アクティブ状態）
- カテゴリーボタン（訪日マーケティングパートナー事業、地域観光DX事業）
- ホバー・アクティブ状態のスタイル

### 3. カードセクション（cards/）

導入事例のカードリスト

- **html.md**: カード要素を含むHTML構造
- **css.md**: PC・タブレット・スマホ全てのブレークポイントに対応したスタイル

**主な要素:**
- カードリスト（ul > li）
- サムネイル画像
- クライアント名
- 記事タイトル
- カテゴリータグ
- ホバーエフェクト

## ブレークポイント

全セクション共通で以下のブレークポイントに対応:

| デバイス | ブレークポイント |
|---------|----------------|
| デスクトップ | デフォルト |
| タブレット | max-width: 1140px |
| タブレット（小） | max-width: 840px |
| スマホ | max-width: 540px |
| スマホ（小） | max-width: 320px |

## 使用方法

1. **html.md** - HTML構造を確認し、WordPressテンプレートでPHP化する
2. **css.md** - CSSスタイルをコピーして `assets/css/case.css` に実装
3. **data-s-*属性** - STUDIOの自動生成属性なので、WordPressでは独自のBEMクラスに置き換える

## 注意事項

- **data-s-*属性**: STUDIO特有の自動生成属性のため、WordPress実装時は削除して独自のクラス名に置き換える
- **CSS変数**: `var(--s-font-*)`, `var(--s-color-*)` などはSTUDIO固有のため、テーマのCSS変数に置き換える
- **画像URL**: `https://storage.googleapis.com/studio-cms-assets/...` は本番環境のURLのため、WordPress管理画面でアップロードした画像パスに変更する

## 実装の流れ

1. HTML構造を確認してWordPressテンプレートを作成
2. CSSスタイルを `case.css` に実装（data-s-*属性 → BEMクラスに変換）
3. レスポンシブ対応（全ブレークポイントで確認）
4. `/review-code` コマンドで最終確認
