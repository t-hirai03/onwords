# 導入事例詳細ページ ドキュメント

本番サイト（https://www.onwords.co.jp/case/retail-poc-strategy）から抽出したHTML構造とCSSスタイルを記録したドキュメント群です。

## 目的

WordPressテーマで導入事例詳細ページ（single-case.php）を実装する際に、本番サイト（STUDIO）と完全に同じ見た目を再現するための参考資料として使用します。

## ディレクトリ構造

```
.docs/case/detail/
├── README.md          # このファイル
├── header/
│   ├── html.md        # 記事ヘッダーのHTML構造
│   └── css.md         # 記事ヘッダーのCSSスタイル
└── content/
    ├── html.md        # 記事本文のHTML構造
    └── css.md         # 記事本文のCSSスタイル
```

## ページ構成

導入事例詳細ページは以下のセクションで構成されています：

### 1. パンくずリスト

ホーム → 導入事例 → 記事タイトル

### 2. ヒーローセクション

背景画像付きのヒーローセクション
- ラベル: "Case"
- タイトル: "導入事例"

### 3. 記事ヘッダー

- タイトル（h1）
- メタ情報
  - クライアント名（アイキャッチ画像 + テキスト）
  - カテゴリ（複数対応、グレー背景）
  - 投稿日（YYYY/MM/DD形式）

### 4. 記事本文

- 大見出し（h2）- 赤下線付き
- 中見出し（h3）- 赤文字
- 小見出し（h4）- 薄い赤文字
- 段落（p）
- リスト（ul/li）- カスタムリストスタイル

### 5. 導入事例一覧へリンク

赤いグラデーションボタン（btn-primary）

## 使用方法

1. **structure.md** - HTML構造を確認し、WordPressテンプレートでPHP化する
2. **styles.md** - CSSスタイルをコピーして `assets/css/case.css` に実装
3. **data-s-*属性** - STUDIOの自動生成属性なので、WordPressでは独自のBEMクラスに置き換える

## 注意事項

- **data-s-*属性**: STUDIO特有の自動生成属性のため、WordPress実装時は削除して独自のクラス名に置き換える
- **CSS変数**: `var(--s-font-*)`, `var(--s-color-*)` などはSTUDIO固有のため、テーマのCSS変数に置き換える
- **画像URL**: 本番環境のURLは、WordPress管理画面でアップロードした画像パスに変更する

## 実装の流れ

1. HTML構造を確認してWordPressテンプレート（single-case.php）を作成
2. CSSスタイルを `case.css` に実装（data-s-*属性 → BEMクラスに変換）
3. レスポンシブ対応（全ブレークポイントで確認）
4. `/review-code` コマンドで最終確認
